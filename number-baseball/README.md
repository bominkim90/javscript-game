## 1단계 : 숫자 야구 게임 구현
```
게임 구현을 설명서 그대로 구현하려함
기능단위로 쪼개는거부터 시작한게 아니라
일단 큰 기능 단위로 함수로 묶었음
그리고 전역변수로 예를들어 nowPlaying이라는 전역변수 flag값을 둬서
콘솔로 들어오는 입력값에 대한 처리방식을 바꾸려함 (이처럼 콘솔 입력값에 대한 처리방식이 바뀌는걸 phase라 칭함)
문제점은 그럼 phase가 바뀔때마다 그것에 대한 flag값을 만들어줘야하므로, 전역변수가 계속 늘어날 가능성이 있음
그래서 2단계에서 수정이 들어감
```


## 2단계 : 게임 기록 저장 및 조회 기능 추가
```
계속해서 늘어나는 기능들로 인해, 함수를 더욱 세분화 함
phase가 계속 늘어남에 따라 flag값을 전역변수로 더 선언하는 방식말고 다른 방식을 찾기로 함
프로그램 시작 시 등록된 이벤트 리스너 => readlineInterface.on("line", first리스너)
기존 리스너 삭제(초기화) => readlineInterface.removeAllListeners("line")
새 등록 => readlineInterface.on("line", new리스너)
위와 같이 "line"이벤트에 대한 리스너를 초기화 하고 
각 phase에 대응하는 각 리스너를 등록함
```

### '순환 참조' 문제발생
```
예를들어
saveGameRecord.js
   ↓ require
lineEventHandler.js
   ↓ require
checkUserNumbers.js
   ↓ require
saveGameRecord.js  ← 다시 돌아옴 (순환 발생)

Node.js에서 모듈을 가져올때 require(파일경로) 를 사용하면
시스템이 그 경로를 찾고 가져오고 연결하는데 시간이 걸림

그런데 처음 파일A 에서 다른 파일B 를 참조하고
B에서 C를 참조하고
마지막엔 C에서 A를 참고하고 이런식으로 서로가 서로를 참조하는 경우가 생기면
NodeJs 모듈시스템에선 require()처리가 시간이 걸려서
서로 참조하는 경우가 있을때 엉킬 수 있다함

그래서 웬만하면 하나의 모듈에는 다른 모듈을 콜백함수로 넣는걸 자제함
그리고 넣는 경우면은 그 모듈은 완전 단순한 형태, 즉
다른 모듈을 참조하지 않고, 어디와도 연결되어있지 않는 형태만 콜백으로 넣던지 함

```
